# VendorBridge — Frontend Handoff & Architecture Guide

This document is designed for the Frontend Engineer and their AI Assistant. It outlines the project architecture, the established API contracts from the backend, and the exact steps required to complete the React frontend using Material UI.

---

## 🏗️ 1. Project Architecture Overview

**VendorBridge** is a Production-Grade Procurement & Vendor Management ERP. 
- **Backend**: FastAPI Modular Monolith + PostgreSQL (Currently being handled by Teammate 1).
- **Frontend**: Vite + React 18 + Material UI (MUI) v5 + React Query + React Router v6.
- **State Management**: `@tanstack/react-query` for server state, `React Context` for global UI/Auth state.
- **Authentication**: JWT (Access & Refresh tokens) via localStorage.

### Core Philosophy
- **Modular Structure**: The frontend `src/modules/` mirrors the backend's domain modules (Auth, Vendors, RFQs, Quotations, Approvals, POs, Invoices, Settings).
- **Premium UI**: We are using a custom dark theme with Material UI. The theme is already configured in `src/theme.js`. Use MUI's `Card`, `DataGrid`, `Dialog`, and `Grid` components extensively.
- **Soft Deletes & Auditing**: The backend uses soft deletes (`is_deleted = True`) for everything. The frontend does not need to handle this explicitly, but be aware that DELETE requests will soft-delete records.

---

## 🤖 2. Instructions for the AI Agent

If you are an AI assisting the Frontend Engineer, follow these strict rules:
1. **Stack**: Write code using Vite, React functional components, hooks, and **Material UI (@mui/material)**. Do NOT use Tailwind CSS or raw CSS files unless absolutely necessary. Use MUI's `sx` prop for styling.
2. **Data Fetching**: Use `axios` (configured in `src/shared/api/client.js`) combined with `@tanstack/react-query` (`useQuery`, `useMutation`).
3. **Routing**: Use `react-router-dom` v6.
4. **Forms**: Use standard controlled React components or a library like `react-hook-form`. Forms must have proper validation and error states using MUI's `TextField` `error` and `helperText` props.
5. **Tables**: Use `@mui/x-data-grid` for complex tables or standard MUI `Table` for simple ones.

## 🔌 3. API Contracts & Data Schema

The backend is hosted at `http://localhost:8000/api/v1`. The Vite proxy is already configured, so make frontend requests to `/api/v1/...`.

### Base Entity (Applies to all models)
Every record returned by the backend includes:
- `id` (UUID)
- `created_on` (Datetime)
- `updated_on` (Datetime)
- `created_by` (UUID)
- `updated_by` (UUID)
- `is_active` (Boolean)

### Core Modules & Endpoints

#### A. Auth (`/auth`)
- `POST /auth/login` → Payload: `{ email, password }` → Returns: `{ access_token, refresh_token, user: { id, full_name, email, role } }`
- `GET /auth/me` → Returns current user based on Bearer token.
- **Roles**: `admin`, `procurement_officer`, `vendor`, `approver`.

#### B. Vendors (`/vendors`)
- `GET /vendors?page=1&size=20` → Returns paginated list.
- `POST /vendors` → Payload: `{ name, category, gst_number, email, phone, address, city, country }`
- **Fields**: `vendor_code` (auto-generated), `status` (active/inactive), `rating` (float 0-5).

#### C. RFQs (Request for Quotation) (`/rfqs`)
- `GET /rfqs` → Returns RFQs.
- `POST /rfqs` → Payload: `{ title, description, deadline, items: [{ item_name, quantity, unit, specifications, estimated_cost }], vendor_ids: [uuid1, uuid2] }`
- `POST /rfqs/{id}/publish` → Transitions from `draft` to `open`.
- **Statuses**: `draft`, `open`, `closed`, `awarded`, `cancelled`.

#### D. Quotations (`/quotations`)
- Vendor facing. Vendors submit pricing for an RFQ.
- `POST /quotations` → Payload: `{ rfq_id, vendor_id, delivery_days, notes, items: [{ rfq_item_id, unit_price }] }`
- `GET /quotations/compare/{rfq_id}` → Critical endpoint for Procurement Officers. Returns a comparison matrix of all vendors, highlighting the lowest price and calculating a "Vendor Score" (40% price, 30% delivery, 30% rating).

#### E. Approvals (`/approvals`)
- `POST /approvals` → Request approval for a selected quotation.
- `GET /approvals/pending` → For Managers to see what needs approval.
- `PUT /approvals/{id}/decide` → Payload: `{ status: "approved" | "rejected", remarks: "string" }`
- **Note**: Approving automatically triggers Purchase Order generation on the backend.

#### F. Purchase Orders & Invoices (`/purchase-orders`, `/invoices`)
- `GET /purchase-orders`
- `GET /invoices`
- `GET /invoices/{id}/pdf` → Returns a binary PDF file generated by the backend.

#### G. Settings (`/settings`)
- `GET /settings` → Returns global Key-Value pairs (e.g., SMTP configurations, company name).
- Editable by Admins to make the ERP dynamic.

## 🚀 4. Current State & Frontend Task List

**What is already built in `frontend/`:**
- `package.json` & `vite.config.js` with proxy.
- `src/theme.js` containing the Premium Dark MUI Theme.
- `src/App.jsx` and `src/main.jsx` with Router and ThemeProvider.
- `src/shared/components/Layout.jsx` featuring an MUI Drawer and AppBar.
- `src/modules/dashboard/DashboardPage.jsx` with MUI StatCards.

### 🎯 Tasks for the Frontend Teammate (in order of priority):

#### Step 1: Authentication Setup
- Build `src/shared/api/client.js` using Axios. Add interceptors to attach the JWT from `localStorage` to the `Authorization: Bearer` header.
- Build `src/modules/auth/AuthContext.jsx` to manage login state.
- Build `src/modules/auth/LoginPage.jsx` using an MUI `Card` and `TextField`s.

#### Step 2: Vendor Management
- Build `VendorsPage.jsx`. Use `@mui/x-data-grid` to display the list of vendors.
- Create an "Add Vendor" `Dialog` with a form.

#### Step 3: RFQ & Procurement Workflow
- Build `RFQsPage.jsx` (List view with MUI Tabs for statuses).
- Build `RFQCreatePage.jsx` (A multi-step MUI `Stepper` form for Basic Info -> Items -> Assign Vendors).
- Build `QuotationComparePage.jsx` (A complex table comparing vendor prices side-by-side, highlighting the winner in green).

#### Step 4: Approvals & Finalization
- Build `ApprovalsPage.jsx` for managers to click "Approve" or "Reject".
- Build `PurchaseOrdersPage.jsx` and `InvoicesPage.jsx`.
- Ensure the "Download PDF" button correctly handles the `blob` response from the `/invoices/{id}/pdf` endpoint.

#### Step 5: Dynamic Settings
- Build `SettingsPage.jsx` with vertical tabs (General, SMTP). Fetch keys from the `/settings` endpoint and allow Admins to update them dynamically.
